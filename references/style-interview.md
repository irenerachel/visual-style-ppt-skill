# 风格保存与访谈模板

用于提炼新视觉风格，并保存为 `styles/{style-name}.md`。

## 提炼输出结构

先输出一版风格提炼草案：

- 风格名称建议。
- 适用场景。
- 视觉关键词。
- 色彩系统。
- 字体与排版气质。
- 图形元素。
- Style Lock：字体层级、版式网格、装饰元素、色彩比例、页脚和禁用项。
- 具体视觉约束：主色/辅色/文字/边框色号、候选字体、边框形态、卡片材质、阴影或发光规则。
- PPT 页面适配。
- 图片提示词骨架。
- 负面约束。
- 质量检查点。

## 访谈问题

只问影响结果的关键问题。优先一轮问完，避免反复打断。

1. 这个风格主要用于什么？
   - PPT 演示
   - 图片版幻灯片
   - 文章配图
   - 小红书/社媒卡片
   - 混合使用

2. 你希望默认比例是什么？
   - 16:9
   - 4:3
   - 1:1
   - 4:5 / 3:4
   - 9:16

3. 文字密度偏好是什么？
   - 极简
   - 中等信息量
   - 信息图密度

4. 输出默认要哪种？
   - 图片序列
   - 图片版 PPTX
   - 图片序列 + 图片版 PPTX

5. 是否保存为可复用风格文件？
   - 如果保存，询问风格名和别名。
   - 如果用户不想命名，根据视觉特征生成稳定英文文件名。

6. 是否有必须固定的品牌元素？
   - Logo
   - 品牌色
   - 字体
   - 水印
   - 作者名

7. 是否允许画面出现日期？
   - 如果允许，必须先确认日期。
   - 如果不允许，风格文件中标注默认不放日期。

8. 下次使用这个 Skill 时，是否默认优先询问调用已有风格模板？
   - 默认建议：是。
   - 如果用户提供新参考图，则询问是提取新风格，还是只把图片作为内容参考。

## 保存格式

风格文件最少包含：

```markdown
# 风格名称

别名：

## 风格定位

## 视觉 DNA

## Style Lock

用于多页图片版 PPT 的跨页控制系统。必须写清：

- 字体气质：主标题、副标题、模块标题、说明文字分别像什么字体。
- 语言规则：默认中文优先，尤其标题、章节名、模块名、结论句和输出文档正文必须优先中文；英文只保留产品名、模型名、品牌名、代码/API 术语和必要专有名词，例如 Claude、Image2、API、XML 标签名。
- 字号层级：H1、H2、模块标题、正文、小字之间的相对比例。
- 版式网格：安全边距、常用布局、页面节奏、标题位置。
- 页面角色：封面可以多强视觉，内页应该多克制；内页的背景和图像如何退后。
- 装饰元素：可重复使用的小元素、几何形、图标、纹理、页脚。
- 风格隔离：明确禁止混入哪些历史参考、其它风格模板、材质、构图或装饰语言。
- 色彩比例：主背景、主色、辅助色、点缀色的大致占比。
- 色号：背景色、主色、辅色、正文色、弱文字色、边框色、强调色、阴影/发光色。
- 边框与容器：必须定义一套统一组件系统，包括直角/圆角、线宽、双线/单线、卡片材质、透明度、阴影或发光方式；同一 deck 不允许每页换一种框架。
- 文本密度：每页最多几个模块、每个模块最多几行；默认低密度，普通内页最多 2-3 个短信息点，每个信息点 1 行。
- 禁用项：这个风格最不能出现的视觉偏移。

## 色彩建议

## PPT 页面适配

## 图片构图规则

## 默认输出参数

## 排版规则

## 信息密度

## 时间与日期规则

## 推荐提示词骨架

## 负面约束

## 质量检查
```

## 调用说明

保存后告诉用户可以这样调用：

```text
用 $visual-style-ppt 调用 {风格名}，把这个文档做成 {页数} 页 PPT
```

```text
用 $visual-style-ppt 调用 {风格名}，生成 {页数} 页图片版幻灯片
```
