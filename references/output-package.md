# 输出包规范

## 最高优先级硬规则：只允许 Image 2 出图

所有图片页、缩略图板、信息图、文章配图和 PPT 页面图片必须由 Image 2 模型生成。严禁用 HTML/CSS、浏览器截图、SVG/HTML mockup、canvas、PIL 或本地脚本输出图片；Image 2 不可用时不得用 HTML 兜底，必须停止并说明需要 Image 2。

## 输出路线

这个 Skill 只能用 Image 2 生成完整图片页，也可以把图片页装进 PPTX。PPTX 只是图片容器，不承担可编辑排版。

### A. 图片序列

适合追求视觉统一、社交传播、文章配图、海报式 PPT、信息图和视觉稿的场景。

制作原则：

- 每一页先按页面类型写 Image2 提示词。
- 每一页生成一张完整视觉图。
- 图片中可以包含短标题、短标签、模块文字，但必须控制文字量。
- 交付 `images/slide-XX.png`，必要时附带 `prompts.md`。

### B. 图片版 PPTX

适合用户需要 PPT 文件，但不需要改字、改图表或编辑对象的场景。

制作原则：

- 先生成图片序列。
- 把每张图片按页面比例全屏铺入 PPTX。
- 交付 `deck.pptx`。这个 PPTX 本质是图片容器，视觉稳定，文字不可编辑。

### C. 最终确认与完整输出包

适合多页项目、可复用项目或后续可能返修的场景。

制作原则：

- Image2 生成完图片后，先展示或列出所有图片给用户确认。
- 用户未确认全部图片前，不创建最终压缩包。
- 用户确认全部图片后，交付一个 `.zip` 压缩包，包含 `deck.pptx`、`images/`、`prompts.md`、`outline.md`、`style-used.md` 和必要的 `revision-log.md`。
- `.zip` 存到用户默认保存文件夹；macOS 默认使用 `~/Downloads`，除非用户指定其他默认保存位置。
- 返修时只重生成目标页图片，再替换 PPTX 对应页。

## 推荐输出目录

```text
outputs/YYYY-MM-DD-project-title/
├── YYYY-MM-DD-project-title.zip # 用户确认全部图片后生成，最终交付压缩包
├── deck.pptx                 # 图片版 PPTX，若用户需要 PPT 文件
├── images/
│   ├── slide-01.png
│   ├── slide-02.png
│   └── ...
├── versions/                  # 返修版本，复杂项目使用
├── prompts.md                 # 每页图片/视觉提示词
├── style-used.md              # 本次使用或提炼的风格文件副本
├── revision-log.md             # 修改记录，返修时使用
└── outline.md                 # 页面大纲、页型、内容分配
```

简单单图任务可以只交付用户明确要求的文件，不必强行生成完整目录。多页 PPT / 多页图片项目默认走最终 `.zip` 包。

## 生成前必须确认

如果用户没有说明，先问：

- 需要多少页？
- 要图片序列、图片版 PPTX，还是两者都要？
- 比例是否用默认 16:9？
- 图片质量是否用默认 2K？
- 是否需要日期、作者、品牌名或水印？Logo/固定字标默认不由 Image2 生成；需要 Logo 时建议用户后期自行添加，或在 PPTX/图片后期用同一个固定资产统一叠加。

若用户说“你来定”，则根据文档长度决定页数和路线，并在计划中说明。

## PPTX 决策规则

- 用户说“做 PPT”“PPTX”“演示文稿”：默认图片版 PPTX。
- 用户说“像图片一样漂亮”“海报式”“小红书”“文章配图”“视觉稿”“信息图”：默认图片序列。
- 用户同时要图片和 PPT：生成图片序列并组装 `deck.pptx`。
- 用户说“可编辑”：说明当前 skill 不做可编辑 PPTX，建议改走普通 presentations workflow；除非用户明确要求仍用图片版。
- 多页项目的最终交付不是散文件，而是在用户确认图片后生成 `.zip`：包含所有最终图片、由图片组成的 PPTX、提示词文档、outline 和风格/修改记录文档。

## 后期修改交付

当用户只修改某一页或某一张图：

- 只重新生成或编辑对应资产。
- 保持其他页面和图片不变。
- 如果有 `deck.pptx`，替换对应页图片后重新导出。
- 如果有 `deck.pptx`，替换对应页图片后重新导出。
- 复杂项目记录到 `revision-log.md`，简单项目在最终回复中说明修改范围。
- 返修完成后仍需等待用户确认全部图片，再重新生成最终 `.zip`。
